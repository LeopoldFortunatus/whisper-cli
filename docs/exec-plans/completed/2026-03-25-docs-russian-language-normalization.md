# План выполнения

Владелец: Platform Team

## Цель

Привести `README.md` и документы в `docs/` к русскому основному языку, оставив English только там, где он нужен как устойчивый technical term, имя команды, API или code identifier.

## Контекст

- В репозитории накопился смешанный стиль: русская основа текста с большим количеством английских заголовков и prose.
- Нужно синхронно обновить документы и `scripts/docs_check.sh`, чтобы quality gate проверял новый стиль.
- Исторические `exec-plan` документы тоже лежат в `docs/` и влияют на общую консистентность.

## Риски

1. Слишком агрессивная русификация может задеть устойчивые technical terms и ухудшить читаемость.
2. Если обновить тексты, но не обновить `docs-check`, локальный quality gate начнёт падать.
3. Если русифицировать только часть `docs/`, в репозитории останется смешанный стиль на уровне шаблонов и архивных планов.

## План

1. Обновить `README.md`, core docs, roadmap и debt tracker.
2. Русифицировать навигационные файлы и шаблоны `exec-plan`, затем привести к тому же стилю completed plans.
3. Обновить `scripts/docs_check.sh` и прогнать `make docs-check` и `make ci`.

## Проверка

- `make docs-check`
- `make ci`

## Журнал решений

- 2026-03-25: русифицировать не только заголовки, но и prose, оставляя English только для устойчивых technical terms и code-level идентификаторов.

## Находки

- Основная неоднородность сидит в заголовках, metadata-полях и prose шаблонов, а не только в отдельных фразах.
- Пришлось отдельно вернуть status-термины в English: `Status`, `Active`, `Completed`, `Done`, `In Progress`, `Planned`, `Blocked`, `Closed`.

## Следующие шаги

- Поддерживать это правило при следующих изменениях документации и не смешивать английские заголовки с русским prose без явной причины.

## Итоги
